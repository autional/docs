# 文档站 UI / 品牌落地规范

> 用途：确保 `autional-docs-site` 后续继续扩展时，保持与主官网一致的品牌风格，同时维护良好的文档可读性。

---

## 1. 核心原则

文档站的首要目标不是“做成营销页”，而是：

- 统一
- 可读
- 清晰
- 有品牌感

任何视觉升级都不能破坏文档的阅读效率。

---

## 2. 品牌令牌

颜色、字体、阴影统一来自：

- `tailwind.config.cjs`

优先复用：

- `primary`
- `sky`
- `amber`
- `shadow-soft`
- `shadow-card`
- `shadow-brand`

避免：

- 新增零散品牌色
- 页面内直接大量硬编码颜色

---

## 3. 全局样式入口

文档站所有新的品牌与正文规范，应优先放入：

- `src/styles/global.css`

当前已提供的关键类包括：

- `brand-shell`
- `brand-card`
- `brand-button-primary`
- `brand-button-secondary`
- `brand-kicker`
- `docs-nav-link`
- `docs-prose`

不要把同一类正文规则重复写进多个页面。

---

## 4. 布局规范

共享布局统一由：

- `src/layouts/DocLayout.astro`

负责承接。

职责分工：

- Header：承担品牌识别
- Sidebar：承担导航
- Main / Article：承担正文展示

后续新增页面不应各自实现独立壳层，优先复用现有布局。

---

## 5. 正文规范

所有文档正文默认进入 `docs-prose` 容器。

正文应保证：

- 标题层级清晰
- 段落行高舒适
- 链接具有可识别性
- 表格可读性高
- 代码块不刺眼也不发灰

如果某页需要特殊排版，优先在 `docs-prose` 基础上做增量，而不是完全脱离它。

---

## 6. 代码块规范

代码块必须满足：

- 对比度足够
- 支持横向滚动
- 不截断命令
- 不把示例代码错误解析为真实 HTML / JSX 节点

如果代码内容含有：

- JSX 标签
- HTML 标签
- XML 风格片段

优先采用普通文本渲染，不使用 `set:html` 直出。

---

## 7. 首页规范

文档站首页应具备：

- 明确的文档门户定位
- 最快上手路径
- 常用入口卡片
- 从“进入文档”到“下一步阅读”的引导

首页不应只是某篇普通文档原样顶上来。

---

## 8. 导航规范

导航应保持：

- 信息清晰
- 当前项高亮
- 跳转路径稳定

后续若做移动端增强，应优先扩展现有导航体系，而不是再做一套独立风格。

---

## 9. 后续维护建议

改动页面前，先确认：

1. 是否复用了已有品牌令牌？
2. 是否进入了 `docs-prose` 正文规范？
3. 代码块是否保证正确渲染？
4. 页面是否仍然以阅读效率为优先？

如果答案大多是否，建议回到本规范重新审视设计。
